## To‑Do List (React + TypeScript + Vite)  

### Быстрый старт
```bash
# Клонировать репозиторий
git clone https://github.com/willalone/todo-list-advanced.git
cd todo-list-advanced

# Установить зависимости
npm install

# Запустить dev-сервер
npm run dev

# Открыть в браузере
# http://localhost:5173
```

#### Доступные команды

```bash
npm run dev          # Запуск dev-сервера с hot-reload
npm run build        # Сборка production-версии в папку dist
npm run preview      # Просмотр production-сборки
npm run typecheck    # Проверка TypeScript типов
npm run lint         # Проверка кода ESLint
npm run format       # Проверка форматирования Prettier
npm run format:write # Авто-форматирование кода
```

### Стек
- React 18, TypeScript
- Vite 5
- Без сторонних стейт-менеджеров — `useReducer` + Context

### Возможности
- Добавление задач с названием, сроком, тегами, приоритетом
- Редактирование, удаление, переключение «выполнено»
- Фильтры по статусу, тегу и приоритету
- Поиск по названию/заметкам/тегам
- Сортировка по дате создания/обновления, названию, сроку, приоритету
- Локальное сохранение в `localStorage`
- Клавиатурная доступность, aria-атрибуты
- Адаптивный интерфейс

### Архитектура

Проект следует современным практикам React и имеет модульную структуру:

#### Компоненты (`src/components/`)
- `TodoApp.tsx` — основной компонент-контейнер
- `TodoStats.tsx` — отображение статистики (активные/завершённые)
- `TodoCreateForm.tsx` — форма создания новой задачи
- `TodoFilters.tsx` — панель фильтров и сортировки
- `TodoList.tsx` — список задач
- `TodoItemRow.tsx` — отдельная задача с встроенным редактированием

#### Кастомные хуки (`src/hooks/`)
- `useTodoFilters.ts` — управление фильтрами и сортировкой

#### Утилиты (`src/utils/`)
- `todoUtils.ts` — pure-функции: создание, обновление, фильтрация, сортировка, получение меток

#### Состояние (`src/state/`)
- `TodoContext.tsx` — Context API с useReducer + персистентность в localStorage

#### Константы (`src/constants/`)
- `defaults.ts` — дефолтные значения, опции для селектов

#### Типы (`src/types.ts`)
- Полная типизация доменной модели

#### Стили (`src/styles.css`)
- Современная адаптивная тема с авто-светлой/тёмной схемой
- CSS-переменные, градиенты, плавные анимации

### Качество кода
- **Строгая типизация** — TypeScript, отсутствие `any`
- **Разделение ответственности** — UI компоненты, бизнес-логика в хуках, pure-функции в utils
- **Производительность** — оптимизация с `useCallback`, `useMemo`
- **Чистый код** — понятные имена, единый стиль, DRY, KISS
- **Константы вместо магических значений** — централизованные опции и дефолты
- **Доступность (a11y)** — aria-атрибуты, семантические роли, управление с клавиатуры
- **CSS-модульность** — переиспользуемые классы, CSS-переменные

### Требования

- Node.js >= 18.x
- npm >= 9.x

### Разработка

Проект настроен с использованием:
- **ESLint** для проверки кода
- **Prettier** для форматирования
- **TypeScript** для типобезопасности
- **GitHub Actions** для CI/CD

Перед коммитом рекомендуется запустить:
```bash
npm run lint          # Проверить код
npm run typecheck     # Проверить типы
npm run format:write  # Отформатировать код
```

### Использование

1. **Добавление задачи**: введите название и нажмите Enter или кнопку "Добавить"
2. **Редактирование**: двойной клик по названию задачи или кнопка "Изменить"
3. **Фильтрация**: используйте панель фильтров для поиска и сортировки задач
4. **Удаление**: нажмите кнопку "Удалить" у задачи
5. **Клавиатура**: Enter для сохранения, Escape для отмены редактирования

### Деплой

Для деплоя соберите production-версию:
```bash
npm run build
```

Файлы будут в папке `dist/`, готовые для хостинга на любом статическом сервере.
